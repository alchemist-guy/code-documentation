1. Introduction to Laravel

What is Laravel?

Laravel is a modern, full-stack PHP framework that provides an elegant syntax and robust tools for building web applications. It follows the Model-View-Controller (MVC) architectural pattern and offers numerous features like routing, authentication, sessions, caching, and more out of the box. Laravel's philosophy emphasizes developer experience and clean, readable code while maintaining flexibility for complex applications.

Key Features

  • Eloquent ORM: Advanced ActiveRecord implementation for working with databases
  • Blade Templating: Lightweight yet powerful templating engine with template inheritance
  • Artisan CLI: Built-in command-line tool for common tasks and code generation
  • Middleware: Flexible system for filtering HTTP requests entering your application
  • Authentication System: Ready-to-use implementation including registration, login, and password reset

MVC Architecture

Laravel follows the Model-View-Controller (MVC) pattern, which separates application logic into three distinct components:

Model

Represents data structure and business logic. Handles database interactions through Eloquent ORM.

View

Presents data to users through Blade templates. Contains HTML/CSS and display logic.

Controller

Handles user requests, processes data from Models, and returns Views.

Basic Example

Here's a simple route example showing Laravel's basic structure: 

                  
// routes/web.php
Route::get('/greeting', function () {
    return view('greeting', ['name' => 'James']);
});

// resources/views/greeting.blade.php
<html>
    <body>
        <h1>Hello, {{ $name }}!</h1>
    </body>
</html>

Code Explanation

  • The route defines a URL endpoint (/greeting) and a closure to handle requests
  • The view() helper returns a Blade template with passed data
  • Blade's {{ }} syntax safely outputs escaped data
  • This demonstrates the MVC flow: Route → Controller-like logic → View

System Requirements

Before installing Laravel, ensure your system meets these requirements:

  • PHP ≥8.1 with extensions:
    • Ctype, cURL, DOM, Fileinfo, Filter, Hash, Mbstring
    • OpenSSL, PCRE, PDO, Session, Tokenizer, XML
  • Composer - PHP dependency manager
  • Database Server (MySQL, PostgreSQL, SQLite, or SQL Server)

3. Routing Basics

3.1 Basic Routing

Routes define entry points to your application and map URLs to controller actions or closures. All Laravel routes are defined in routes/web.php for web routes and routes/api.php for API routes. 


// Basic GET route
Route::get('/welcome', function () {
    return view('welcome');
});

// Supporting multiple HTTP verbs
Route::match(['get', 'post'], '/contact', function () {
    // Handle both GET and POST requests
});

// All HTTP verbs
Route::any('/any-request', function () {
    // Handle any HTTP method
});

💡 Route Types

  • Web Routes: For UI interactions (session state, CSRF protection)
  • API Routes: For stateless JSON APIs (rate limiting, auth via tokens)
  • Console Routes: For custom Artisan commands
  • Channel Routes: For broadcasting events via WebSockets

3.2 Route Parameters

Routes can capture segments of the URI as parameters. Parameters are always enclosed in curly braces {} and passed to your route callback/controller.

Required Parameters


Route::get('/user/{id}', function ($id) {
    return "User ID: ".$id;
});

Parameters must be present in the URL. Missing parameters will trigger 404.

Optional Parameters


Route::get('/posts/{slug?}', function ($slug = 'latest') {
    return "Showing post: ".$slug;
});

Mark parameters optional with ?. Provide default values.

🔍 Parameter Constraints 


Route::get('/user/{id}', function ($id) {
    // ...
})->where('id', '[0-9]+');

// Multiple constraints
Route::get('/post/{year}/{slug}', function ($year, $slug) {
    // ...
})->where([
    'year' => '[0-9]{4}',
    'slug' => '[A-Za-z\-]+'
]);

Use regular expressions to validate parameters before they reach your route logic

3.3 Named Routes

Named routes allow convenient URL generation and redirects. Names should be unique. 


// Assigning name
Route::get('/dashboard', function () {
    // ...
})->name('dashboard');

// Generating URLs
$url = route('dashboard');

// Redirecting
return redirect()->route('dashboard');

✅ Benefits

  • Decouple URLs from implementation
  • Automatic URL updates when routes change
  • Easier maintenance in large applications

📌 Parameter Handling 


// With parameters
Route::get('/user/{id}/profile', function ($id) {
    // ...
})->name('profile');

// Generate URL with parameter
route('profile', ['id' => 12]);

3.4 Route Groups

Route groups allow you to apply attributes to multiple routes without repeating code.

Middleware Groups 


Route::middleware(['auth', 'verified'])->group(function () {
    Route::get('/dashboard', function () {
        // Uses auth + verified middleware
    });
    
    Route::get('/settings', function () {
        // Uses auth + verified middleware
    });
});

Prefix Groups 


Route::prefix('admin')->group(function () {
    Route::get('/users', function () {
        // Matches /admin/users
    });
    
    Route::get('/settings', function () {
        // Matches /admin/settings
    });
});

Name Prefixes 


Route::name('admin.')->group(function () {
    Route::get('/users', function () {
        // Route name: admin.users
    })->name('users');
});

3.5 Route Model Binding

Laravel automatically injects model instances into your routes that type-hint Eloquent models.

Implicit Binding


Route::get('/user/{user}', function (User $user) {
    return $user;
});

Automatically retrieves User model matching the route parameter (ID or slug)

Custom Keys


// Use username instead of ID
Route::get('/user/{user:username}', function (User $user) {
    return $user;
});

Specify which database column to use for model resolution

⚠️ Important Notes

  • Models must type-hint the route parameter variable
  • 404 is automatically thrown if model not found
  • Works with controller methods as well
  • Supports soft-deleted models with withTrashed()

4. Controllers

4.1 Controller Basics

Controllers group related request handling logic in a single class. They serve as intermediary between Routes and Views/Models, following the Single Responsibility Principle.

Key Responsibilities

  • Process incoming HTTP requests
  • Retrieve data from Models
  • Return appropriate responses
  • Handle validation and errors
  • Manage application flow

Lifecycle Flow

Route → Controller → Model (Database) → View → Response

Middleware can intercept at any stage

4.2 Creating Controllers 


php artisan make:controller UserController

6. Models & Eloquent ORM

6.1 Eloquent Fundamentals

Eloquent is Laravel's advanced ActiveRecord implementation that provides an elegant way to interact with databases. Each database table has a corresponding "Model" that allows you to query data and insert/update records. Eloquent models use conventions to minimize configuration while providing powerful query capabilities.

Key Features

  • Automatic database table mapping
  • Built-in CRUD operations
  • Relationship management
  • Mass assignment protection
  • Database agnostic (supports MySQL, PostgreSQL, SQLite, SQL Server)

Model Conventions

  • Class name in singular PascalCase (User => users table)
  • Primary key named id (auto-incrementing integer)
  • Timestamps created_at and updated_at
  • Models stored in app/Models directory

6.2 Creating Models 


# Create model with migration
php artisan make:model Product -m

Basic Model Structure 


namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Product extends Model
{
    /**
      * The attributes that are mass assignable
      * @var array
      */
    protected $fillable = [
        'name', 
        'price',
        'description'
    ];

    /**
      * The attributes that should be hidden for arrays
      * @var array
      */
    protected $hidden = [
        'internal_code'
    ];

    /**
      * The attributes that should be cast to native types
      * @var array
      */
    protected $casts = [
        'price' => 'decimal:2',
        'is_active' => 'boolean'
    ];
}

Code Explanation

  • $fillable: Specifies which attributes can be mass assigned (security measure)
  • $hidden: Attributes excluded from array/JSON representations
  • $casts: Automatically converts attributes to common data types
  • Timestamps automatically managed by Eloquent

6.3 CRUD Operations

Create


// Create and save immediately
$product = Product::create([
    'name' => 'Laptop',
    'price' => 999.99
]);

// Create instance then save
$product = new Product;
$product->name = 'Monitor';
$product->save();
Mass Assignment: Requires fields to be in $fillable

Read


// Get all records

// Find by ID
$product = Product::find(1);

// First matching record
$product = Product::where('price', '>', 500)
    ->orderBy('name')
    ->first();
Eager Loading: Use with() to prevent N+1 queries

Update


// Update single instance
$product = Product::find(1);
$product->price = 899.99;
$product->save();

// Mass update
Product::where('category', 'electronics')
    ->update(['discount' => 0.15]);

Delete


// Delete instance
$product = Product::find(1);
$product->delete();

// Direct deletion
Product::destroy(1);
Product::where('price', '>', 1000)->delete();
Soft Deletes: Use SoftDeletes trait for archive functionality

6.4 Eloquent Relationships

Eloquent relationships define connections between models. They serve as powerful query builders while providing intuitive ways to access related data.

One-to-Many

{

// Usage

7. Database Migrations

7.1 Migration Fundamentals

Migrations are version control for your database schema - they allow teams to modify and share the application's database structure consistently. Each migration file contains two main methods: up (apply changes) and down (revert changes).

Key Benefits

  • Team-wide schema consistency
  • Database-agnostic schema definitions
  • Rollback capabilities for schema changes
  • Track schema changes alongside code
  • Environment-specific database setups

File Structure


2023_08_20_000000_create_users_table.php
↑           ↑         ↑
|           |         └── Migration Name
|           └── Month
└── Year

Timestamp format: YYYY_MM_DD_HHMMSS (ensures chronological order)

7.2 Creating Migrations 


php artisan make:migration create_posts_table
php artisan make:migration add_category_to_posts_table --table=posts

Basic Migration Structure 


use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

class CreateUsersTable extends Migration
{
    public function up()
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id(); // Auto-incrementing primary key
            $table->string('name');
            $table->string('email')->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken();
            $table->timestamps(); // created_at & updated_at
        });
    }

    public function down()
    {
        Schema::dropIfExists('users');
    }
}

Code Breakdown

  • Schema::create - Creates new database table
  • Blueprint $table - Defines table structure
  • ->id() - Creates auto-incrementing primary key
  • ->string() - VARCHAR equivalent column
  • ->unique() - Adds unique index constraint
  • ->timestamps() - Adds standard timestamp columns
  • down() - Defines rollback behavior

Column Types


// Common column types
$table->string('title', 100);    // VARCHAR(100)
$table->text('description');     // TEXT
$table->integer('views');        // INTEGER
$table->float('price');          // FLOAT
$table->decimal('total', 8, 2);  // DECIMAL(8,2)
$table->boolean('active');       // BOOLEAN/TINYINT
$table->dateTime('published_at');// DATETIME

Full list of 25+ column types in Laravel documentation

Indexes & Constraints


// Single column index
$table->string('email')->unique();

// Composite index
$table->index(['state', 'country']);

// Foreign key constraint
$table->foreignId('user_id')
      ->constrained()
      ->onUpdate('cascade')

Indexing Best Practices

  • Index columns used in WHERE, JOIN, ORDER BY clauses
  • Avoid over-indexing - each index slows writes
  • Use composite indexes for multi-column queries
  • Consider full-text indexes for search columns
  • Regularly analyze query performance

7.6 Security Considerations

Sensitive Data

// Better approach - use environment variables

  • Use database encryption for sensitive fields

  • Regularly audit database permissions
  • Use migration groups for multi-tenant systems
  • Test rollback procedures regularly

8. Middleware

8.1 Middleware Fundamentals

Middleware acts as a series of layered filters for HTTP requests, providing a convenient mechanism for inspecting and filtering requests entering your application. They execute in the order they're registered, wrapping around your application logic like concentric layers of an onion.

Key Concepts

  • Authentication: Verify user identity before granting access
  • CSRF Protection: Prevent cross-site request forgery attacks
  • Request Modification: Alter incoming requests before handling
  • Response Modification: Modify outgoing responses
  • Logging: Track request/response details

Request Lifecycle

HTTP Request Received
Global Middleware
Route Middleware
Controller Logic
Response Middleware
HTTP Response Sent

8.2 Creating Middleware 


php artisan make:middleware EnsureUserIsAdmin

Middleware Structure 


namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class EnsureUserIsAdmin
{
    /**
      * Handle incoming request
      * @param  Request  $request
      * @param  Closure  $next
      * @return mixed
      */
    public function handle(Request $request, Closure $next)
    {
        if (!auth()->user()->is_admin) {
            abort(403, 'Administrative access required');
        }

        return $next($request);
    }
}

Code Breakdown

  • handle(): Required method that processes the request
  • $next: Closure to pass request to next middleware
  • abort(403): Helper to return HTTP error response
  • auth()->user(): Access authenticated user instance
  • Must return either a response or $next($request)

8.3 Middleware Types

Global Middleware


// app/Http/Kernel.php
protected $middleware = [
    \App\Http\Middleware\TrustProxies::class,

    \App\Http\Middleware\PreventRequestsDuringMaintenance::class,

    \Illuminate\Foundation\Http\Middleware\ValidatePostSize::class,
    // ...
];
Runs on every request to the application. Use sparingly for essential functions like maintenance mode and request validation.

Route Middleware


// app/Http/Kernel.php
protected $routeMiddleware = [
    'auth' => \App\Http\Middleware\Authenticate::class,

    'admin' => \App\Http\Middleware\EnsureUserIsAdmin::class,

    'throttle' => \Illuminate\Routing\Middleware\ThrottleRequests::class,
];

// routes/web.php
Route::get('/admin/dashboard', function () {
    // ...
})->middleware(['auth', 'admin']);
Applied to specific routes or groups. Allows granular control over which requests get processed through which middleware.

8.4 Middleware Parameters 


// Middleware definition
public function handle($request, $next, $role)
{
    if (!auth()->user()->hasRole($role)) {
        abort(403);
    }
    return $next($request);
}

// Route usage
Route::get('/reports', function () {
    // ...
})->middleware('role:manager');

Parameter Handling Details

  • Parameters passed after middleware name separated by colons
  • Multiple parameters: middleware('check:owner,edit')
  • Parameters available in handle method after $request and $next
  • Always validate/sanitize parameters before use

8.5 Middleware Groups

11.1 Cache Fundamentals

Laravel's cache system provides a unified API for various caching backends. It helps improve application performance by storing frequently accessed data in fast storage, reducing database load and computation time.

Key Concepts

  • Drivers: Supported backends (file, redis, memcached, database)
  • TTL: Time-to-Live (cache expiration time)
  • Tags: Categorize related cache items
  • Atomic Locks: Prevent cache stampedes
  • Cache Levels: Object, query, and fragment caching

Explanation: This configuration:

  • Uses file caching by default

11.2 Basic Cache Operations

Storing Items

$value = Cache::pull('key');

    // ...
}

  • Avoid unnecessary has() checks - use get() with default

11.4 Advanced Caching Patterns

Atomic Locks


$lock = Cache::lock('resource:update', 10);

if ($lock->get()) {
    try {
        // Update shared resource
    } finally {
        $lock->release();
    }
}

Prevents:

  • Cache stampedes
  • Race conditions
  • Duplicate processing

Multi-tier Caching


$value = Cache::remember('key', $ttl, function () {
    // Check in-memory cache first
    if ($memoryCache->has('key')) {
        return $memoryCache->get('key');
    }
    
    // Then check Redis
    $value = Redis::get('key');
    $memoryCache->put('key', $value);
    
    return $value;
});

  1. In-memory (APCu)
  2. Redis/Memcached
  3. Database

13. Events & Listeners

13.1 Event Fundamentals

Events provide a simple observer implementation for decoupled application components. They allow application components to communicate without direct dependencies, following the publish-subscribe pattern.

Key Concepts

  • Events: Class-based representations of occurrences in your application
  • Listeners: Classes that handle specific events
  • Dispatcher: Manages event distribution to listeners
  • Queued Listeners: Handle events asynchronously
  • Subscribers: Classes that handle multiple events

Event Lifecycle

1. Event occurs in application
2. Event is dispatched
3. All registered listeners receive event
4. Listeners process event sequentially
5. Application continues execution

14.5 CI/CD Pipeline


# .github/workflows/deploy.yml
name: Deploy Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code
      uses: actions/checkout@v2
    
    - name: Setup PHP
      uses: shivammathur/setup-php@v2
      with:
        php-version: '8.1'
        extensions: mbstring, ctype, fileinfo, openssl, PDO, mysql, pgsql, redis
    
    - name: Install dependencies
      run: composer install --no-dev --prefer-dist --optimize-autoloader
    
    - name: Run tests
      run: php artisan test
    
    - name: Deploy to production
      uses: appleboy/ssh-action@v0.1.10
      with:
        host: ${{ secrets.SSH_HOST }}
        username: ${{ secrets.SSH_USER }}
        key: ${{ secrets.SSH_KEY }}
        script: |
          cd /var/www/project
          git pull origin main
          php artisan migrate --force
          npm run prod

14.7 Security Practices


# File permissions

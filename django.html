1. Introduction to Django

What is Django?

Django is a high-level Python web framework that enables rapid development of secure and maintainable websites. Following the "Batteries-Included" philosophy, it provides built-in features for common web development tasks.

Key Features

  • Batteries-Included Approach:

    Comes with authentication, ORM, admin interface, form handling, and security features out-of-the-box

  • MVT Architecture:

    Model-View-Template pattern separates data handling, business logic, and presentation layer

    
# Example of MVT components:
Model → Database structure
View → Business logic
Template → HTML presentation
  • DRY Principle:

    "Don't Repeat Yourself" philosophy encourages code reuse and reduces redundancy

Why Choose Django?

Rapid Development

Built-in admin interface and ORM allow quick prototyping and iteration

Security

Automatic protection against common vulnerabilities like SQL injection and XSS

Scalability

Used by high-traffic sites like Instagram and Pinterest

Community

Large ecosystem with 4,000+ packages and extensive documentation

2. Installing Django

2. Virtual Environment Setup

A virtual environment isolates project dependencies. This prevents version conflicts between different Django projects. We recommend using Python's built-in venv module:

Create Environment


# Create project directory
mkdir myproject
cd myproject

# Create virtual environment
python -m venv venv

Activate Environment


# Linux/macOS
source venv/bin/activate

# Windows
venv\Scripts\activate
Note: Your prompt should show (venv) when activated. To exit later, run deactivate.

With the virtual environment activated, install Django using pip: 


pip install django

4.3 Database Relationships

Foreign Key (One-to-Many)


class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(
        Author,
        on_delete=models.CASCADE,
        related_name='books'
    )

Explanation: Each book has one author, an author can have many books.
on_delete=models.CASCADE deletes books when author is deleted

Many-to-Many


class Tag(models.Model):
    name = models.CharField(max_length=50)

class Article(models.Model):
    title = models.CharField(max_length=200)
    tags = models.ManyToManyField(Tag)

Creates a join table automatically. Articles can have multiple tags, tags can be associated with multiple articles

4.4 Model Methods


class Order(models.Model):

        self.is_paid = True
        self.save(update_fields=['is_paid'])

    @property
    def tax_amount(self):

    def __str__(self):

5. Django Views

5.1 View Fundamentals

Views are the core of Django's request-response cycle. They:

Basic Responsibilities

  • Receive HTTP requests
  • Process data through models
  • Return HTTP responses (HTML, JSON, etc.)
  • Handle business logic

Function-Based View


from django.http import HttpResponse
from .models import Product

def product_list(request):
    # Get all products from database
    products = Product.objects.all()
    
    # Return simple text response
    return HttpResponse("Listing all products")

This basic view retrieves products from the database using the ORM and returns a plain text response. The request parameter contains all incoming request data like headers and user session.

5.2 Response Handling

Common Response Types

def example_responses(request):
    # Return HTML template

    # Return JSON data
    json_response = JsonResponse({'status': 'success'})
    
    # Redirect to another URL

Template Responses: Use render() for HTML output
API Responses: JsonResponse for REST APIs
Redirects: Maintain proper navigation flow

5.3 Class-Based Views

Basic Class-Based View


from django.views import View
from django.shortcuts import render
from .models import Product

class ProductListView(View):
    template_name = 'products/list.html'

    def get(self, request):
        products = Product.objects.all()
        return render(request, self.template_name, 
                  {'products': products})

    def post(self, request):
        # Handle product creation
        return redirect('product_list')

Class-based views separate request methods (GET, POST) into individual class methods. This makes code more organized for complex views handling multiple operations.

Generic Class Views


from django.views.generic import ListView, DetailView

class ProductListView(ListView):
    model = Product
    template_name = 'products/list.html'
    context_object_name = 'products'

class ProductDetailView(DetailView):

Built-in Generic Views:
- ListView: Display object lists

- CreateView/UpdateView: Handle forms

6. Django URL Configuration

6.1 URL Pattern Basics

What are URL Patterns?

URL patterns are the roadmap of your Django application. They define:

  • Which view handles each URL
  • How to extract parameters from URLs
  • URL naming for reverse resolution
  • Application routing structure

# project/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('articles/2023/', views.special_case_2023),
    path('articles/<int:year>/', views.year_archive),
    path('articles/<int:year>/<slug:slug>/', views.article_detail),
]

6.2 Path Converters

Built-in Converters


path('user/<str:username>/', views.user_profile),
path('post/<int:post_id>/', views.post_detail),
path('slug/<slug:slug>/', views.slug_view),
path('uuid/<uuid:uuid>/', views.uuid_view),
path('path/<path:subpath>/', views.path_view),

Converter Types:
- str: Any non-empty string (excluding '/')
- int: Positive integers
- slug: ASCII letters, numbers, hyphens, and underscores
- uuid: Universally unique identifier format
- path: Any string including '/' characters

6.3 Regular Expression URLs

Advanced Pattern Matching


from django.urls import re_path

urlpatterns = [
    re_path(
        r'^articles/(?P<year>[0-9]{4})/$',
        views.year_archive
    ),
    re_path(
        r'^articles/(?P<year>[0-9]{4})/(?P<month>[0-9]{2})/$',
        views.month_archive
    ),
]

Regex Groups:
- (?P<name>pattern): Named capture group
- [0-9]{4}: Exactly 4 digits
- ^: Start of string anchor
- $: End of string anchor

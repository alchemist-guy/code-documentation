1. Introduction to Ruby on Rails

What is Ruby on Rails?

Rails is a full-stack web framework that provides:

  • Convention over Configuration: Sensible defaults for rapid development
  • DRY Principle: "Don't Repeat Yourself" philosophy
  • MVC Architecture: Clean separation of concerns
  • Built-in Testing: Test-driven development support

üç∞ Cake Baking Analogy

Think of Rails as a professional kitchen where:

  • Chef = Rails Developer
  • Recipe Book = Rails Conventions
  • Ingredients = Gems (Libraries)

üìö Historical Context

Created by David Heinemeier Hansson in 2004, Rails has powered major platforms like:

  • GitHub
  • Shopify
  • Airbnb (initial version)

Complete Workflow Example

  1. User visits /users/1
  2. Routes ‚Üí UsersController#show

  4. Model queries database
  5. Controller passes user to View
  6. View renders HTML response

4. Routing Fundamentals

What are Routes?

Routes are URL patterns that map to controller actions. They work like:

üìÆ Post Office Analogy:

  • URL = Mailing address
  • Router = Postal sorting system
  • Controller = Recipient processing mail

        # config/routes.rb
        Rails.application.routes.draw do
          get '/about', to: 'pages#about'
          get '/contact', to: 'pages#contact'
        end

RESTful Resource Routing

Standard CRUD routes for a resource:

HTTP Verb Path Controller#Action
GET /posts posts#index
GET /posts/new posts#new

üöÄ Quick Setup

Generate all REST routes with one line:


resources :posts  # Creates all 7 RESTful routes

Advanced Routing Techniques

API Namespacing


        namespace :api do
          namespace :v1 do
            resources :users
          end
        end
        # Creates /api/v1/users routes

Member Routes


        resources :posts do
          member do
            get 'preview'
          end
        end
        # /posts/1/preview

Collection Routes


        resources :posts do
          collection do
            get 'search'
          end
        end
        # /posts/search

5. Controller Implementation

What Do Controllers Do?

üéÆ Controller Responsibilities:

  • Process incoming requests
  • Interact with models
  • Prepare data for views
  • Handle response formats (HTML/JSON)

      # app/controllers/posts_controller.rb
      class PostsController < ApplicationController
        def index
          @posts = Post.all
        end
      
        def show
          @post = Post.find(params[:id])
        end
      end

Secure Data Handling

üîí Never Do This:


# UNSAFE - Allows mass assignment
Post.new(params[:post])

‚úÖ Safe Approach (Strong Parameters):


private

def post_params
params.require(:post).permit(:title, :content, :category_id)
end
              

def create
@post = Post.new(post_params)
# ...
end

Controller Filters


    before_action :set_post, only: [:show, :edit, :update, :destroy]
    
    private
    
    def set_post
      @post = Post.find(params[:id])
    end

before_action

Runs before specified actions

after_action

Runs after specified actions

around_action

Wraps around actions

Format Responses


def show
@post = Post.find(params[:id])

respond_to do |format|
  format.html # show.html.erb
  format.json { render json: @post }
  format.pdf do
    render pdf: "post_#{@post.id}", template: 'posts/show'
  end
end
end

üì¢ Common HTTP Status Codes:

  • 200 OK - Success
  • 404 Not Found - Missing resource
  • 422 Unprocessable Entity - Validation errors

6. Views & ERB Templates

Understanding ERB

üìù ERB = Embedded Ruby

  • <% %> Executes Ruby code
  • <%= %> Outputs Ruby result
  • <%# %> ERB comments

        <%# app/views/posts/index.html.erb %>
        <h1 class="text-2xl font-bold mb-4"><%= @user.name %>'s Posts</h1>
        
        <% @posts.each do |post| %>
          <div class="post-card">
            <h2><%= post.title %></h2>
            <p><%= truncate(post.content, length: 150) %></p>
          </div>
        <% end %>

üß† What This Does:

  • Line 2: Displays the user's name from controller's @user instance variable
  • Line 4: Starts loop through @posts collection
  • Line 5-7: Creates a card for each post with truncated content
  • <%= %> Outputs values to HTML
  • <% %> Executes logic without output

Partial Templates


<%# app/views/posts/_form.html.erb %>
<%= form_with model: @post do |form| %>
<div class="field">
  <%= form.label :title %>
  <%= form.text_field :title, class: "input" %>
</div>
<% end %>
                  

<%# app/views/posts/new.html.erb %>
<h1>New Post</h1>
<%= render 'form' %>

üß† Key Concepts:

  • Partial Files: Start with underscore _form.html.erb
  • form_with: Rails helper creating <form> tag with proper attributes
  • render 'form': Reuses the partial template
  • @post: Instance variable passed from controller

Application Layout


<%# app/views/layouts/application.html.erb %>
<!DOCTYPE html>
<html>
<head>
  <title>MyApp</title>
  <%= csrf_meta_tags %>
</head>
<body class="bg-gray-50">
  <%= yield %> <!-- Page content injects here -->
</body>
</html>

üîç Explanation:

  • csrf_meta_tags: Security feature preventing CSRF attacks
  • yield: Placeholder where individual view content is inserted
  • Layouts: Wrap all views for consistent header/footer

Built-in View Helpers

Link Helpers


    <%= link_to "View Post", post_path(@post), 
        class: "btn-primary", 
        data: { turbo_method: :delete } %>

Generates:

<a href="/posts/1" class="btn-primary" data-turbo-method="delete"> View Post </a>

Turbo Drive handles DELETE request asynchronously

Form Helpers


    <%= form_for @user do |f| %>
      <%= f.text_field :email, placeholder: "Enter email" %>
      <%= f.submit "Save", class: "btn-submit" %>
    <% end %>

Generates:

<form action="/users" method="post"> <input type="text" name="user[email]" placeholder="Enter email"> <button type="submit" class="btn-submit"> Save </button> </form>

Creating Custom Helpers


    # app/helpers/posts_helper.rb
    module PostsHelper
      def formatted_date(date)
        date.strftime("%B %d, %Y") #=> "January 01, 2023"
      end
    end

üí° Usage:

  • Available in all views after creation
  • Call with: <%= formatted_date(@post.created_at) %>
  • Keeps complex logic out of views

Managing Assets


<%# In layout file %>
<%= stylesheet_link_tag "application", "data-turbo-track": "reload" %>
<%= javascript_include_tag "application", "data-turbo-track": "reload", defer: true %>

üì¶ What This Means:

  • stylesheet_link_tag: Links compiled CSS from app/assets/stylesheets
  • javascript_include_tag: Loads JS from app/assets/javascript
  • data-turbo-track: Auto-reloads assets when changed
  • defer: Loads JavaScript after HTML parsing

1. Introduction to React

What is React?

React is a JavaScript library for building user interfaces, developed by Facebook (now Meta) in 2013. Unlike full-fledged frameworks, React focuses specifically on the view layer, offering:

  • Component-based architecture
  • Declarative programming paradigm
  • Efficient DOM updates through Virtual DOM
  • Unidirectional data flow

Key Features

1. Components

React applications are built using reusable components - self-contained pieces of code that manage their own state and UI. 


function Greeting() {
  return <h1>Hello, World!</h1>;
}

2. JSX (JavaScript XML)

Syntax extension that allows writing HTML-like code in JavaScript files. 


const element = <div className="app">
  <Welcome name="Sarah" />
</div>;

3. Virtual DOM

React maintains a lightweight copy of the actual DOM. When state changes occur:

  1. Creates new Virtual DOM snapshot
  2. Diffs it with previous version (reconciliation)
  3. Updates real DOM only where necessary

Why Choose React?

Advantages

  • ⚡ High performance with Virtual DOM
  • ♻️ Strong reusability with components
  • 📚 Rich ecosystem (React Router, Redux, etc.)
  • 📱 Cross-platform (React Native)

Use Cases

  • SPAs (Single Page Applications)
  • Data-intensive dashboards
  • Progressive Web Apps
  • Complex UI systems

First React Component

Let's create a simple functional component and understand its structure: 


import React from 'react';

function App() {
  return (
    <div className="container">
      <h1>Welcome to Coderz!</h1>

3. JSX Deep Dive

What is JSX?

JSX (JavaScript XML) is a syntax extension for JavaScript that allows writing HTML-like markup within JavaScript files. It provides syntactic sugar for React.createElement() calls, enabling developers to describe UI structure more intuitively.

JSX Example


const element = (
  <div className="greeting">
    <h1>Hello, {name}!</h1>
    <p>Welcome to Coderz Marketplace</p>
  </div>
);

Compiled JavaScript 


const element = React.createElement(
  'div',
  { className: 'greeting' },
  React.createElement('h1', null, `Hello, ${name}!`),
  React.createElement('p', null, 'Welcome to Coderz Marketplace')
);

JSX Core Rules

1. Single Parent Element

JSX expressions must have a single root element. Use fragments <></> when needing multiple siblings: 


// Invalid ❌
return (
  <h1>Header</h1>
  <p>Content</p>
);
                      

// Valid ✅
return (
  <>
    <h1>Header</h1>
    <p>Content</p>
  </>
);

2. JavaScript Expressions

Embed JavaScript expressions using curly braces { }: 


// Variables
const user = { name: 'Alice' };
<div>Hello, {user.name}</div>
                      

// Expressions
<div>
  {Math.random() * 100}
  {tasks.filter(task => task.completed).length}
</div>

⚠️ Important Notes:

  • Cannot use statements (if/for) directly in JSX
  • Undefined/null/booleans render nothing
  • Objects cannot be rendered directly

6. State Management Fundamentals

6.1 useState Hook

The useState hook in React is a tool that allows you to manage and update data (called state) in your components. Think of state as a container that holds information that can change over time, like user input, a counter value, or fetched data.

Basic Usage 


import { useState } from 'react';

function Counter() {
  // Declare state variable 'count' with initial value 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      {/* Update state with updater function */}
      <button onClick={() => setCount(prev => prev + 1)}>
        Increment
      </button>
    </div>
  );
}

Key Points:

  • useState returns array with [currentState, setterFunction]
  • Always use the setter function to update state
  • State updates are asynchronous and batched
  • Functional updates (prev => newValue) ensure latest state

Complex State 


function UserForm() {
  const [user, setUser] = useState({
    name: '',
    email: '',
    preferences: {
      theme: 'light',
      notifications: true
    }
  });

  const handleChange = (e) => {
    setUser(prev => ({
      ...prev,
      [e.target.name]: e.target.value
    }));
  };

  return (
    <form>
      <input
        name="name"
        value={user.name}
        onChange={handleChange}
      />
      {/* Nested state update */}
      <button onClick={() => {
        setUser(prev => ({
          ...prev,
          preferences: {
            ...prev.preferences,
            theme: 'dark'
          }
        }));
      }}>
        Dark Mode
      </button>
    </form>
  );
}

Best Practices:

  • Always spread previous state when updating objects
  • Use nested spreading for deep object updates
  • Consider useReducer for complex state logic

6.2 Side Effects with useEffect

The useEffect hook in React is used to manage side effects in your components. Side effects are actions that occur outside the normal flow of rendering, such as: ✅ Fetching data from an API ✅ Subscribing to events ✅ Updating the DOM directly ✅ Setting or clearing timers

API Fetching Example 


function ProjectList() {
  const [projects, setProjects] = useState([]);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    const fetchProjects = async () => {
      setLoading(true);
      try {
        const response = await fetch('/api/projects');
        const data = await response.json();
        setProjects(data);
      } catch (error) {
        console.error('Fetch error:', error);
      } finally {
        setLoading(false);
      }
    };

    fetchProjects();
  }, []); // Empty dependency array = run once

  return (
    <div>
      {loading ? (
        <Spinner />
      ) : (
        projects.map(project => (
          <ProjectCard key={project.id} data={project} />
        ))
      )}
    </div>
  );
}

Dependency Array Rules:

  • []: Run once on mount
  • [var1, var2]: Run when variables change
  • No array: Run on every render

Cleanup Function 


function TimerComponent() {
  const [time, setTime] = useState(0);

  useEffect(() => {
    const intervalId = setInterval(() => {
      setTime(prev => prev + 1);
    }, 1000);

    // Cleanup: Remove interval on unmount
    return () => clearInterval(intervalId);
  }, []);

  return <div>Elapsed time: {time}s</div>;
}

When to Cleanup:

  • Event listeners
  • Subscriptions
  • Timers/intervals
  • API request cancellations

6.3 State Sharing Patterns

Lifting State Up is a React pattern used to share state between multiple components by moving the shared state to their common parent component. This ensures both child components can access and update the same data.

Why Use Lifting State Up? When two or more components need to share data (e.g., a form input and a display area), it's better to store the state in their closest shared parent instead of duplicating state in each child.

Lifting State Up 


function ParentComponent() {
  const [sharedState, setSharedState] = useState('');

  return (
    <>
      <ChildA 
        state={sharedState}
        setState={setSharedState}
      />
      <ChildB 
        state={sharedState}
        setState={setSharedState}
      />
    </>
  );
}

function ChildA({ state, setState }) {
  return (
    <input
      value={state}
      onChange={(e) => setState(e.target.value)}
    />
  );
}

When to Use:

  • Direct parent-child relationships
  • Small applications
  • Simple state sharing needs

useReducer Hook 


const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    default:
      throw new Error();
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <>
      Count: {state.count}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
    </>
  );
}

When to UseReducer:

  • Complex state logic
  • Large state objects
  • Predictable state transitions
  • Easier testing of state changes

6.4 State Management Pitfalls

Direct Mutation 


// Wrong ❌
const [user, setUser] = useState({ name: 'John' });
user.name = 'Doe'; // Direct mutation
setUser(user);

// Correct ✅
setUser(prev => ({ ...prev, name: 'Doe' }));

Never mutate state directly - always create new objects/arrays

Stale Closures 


function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const id = setInterval(() => {
      // Stale count value
      setCount(count + 1);
    }, 1000);

    return () => clearInterval(id);
  }, []); // Missing dependency

  // Fix: Use functional update
  setCount(prev => prev + 1);
}

Always use functional updates when state depends on previous value

  • Preventing regression in complex UIs
  • Documenting component variations
